The AI assistant — how it works

The Workbench hands a language model the computed, cited reading and asks it to explain the result in plain language, narrate a Hermetic Codex of the Hour, or plan a historical working using the engine tools. Because the engine numbers are computed and passed in, the model narrates real results rather than inventing them.

Honest note The assistant narrates a historical, pseudoscientific tradition. A locked system prompt holds it to the site's framing: it describes Lilly's and the Picatrix's reasoning, it does not advise, predict, or present astrology as valid. Magical/talismanic content is historical only. See Sources & Science.

Claude (the Anthropic API) — the assistant backend

The assistant calls Claude directly from your browser using your own Anthropic API key:

  1. Get an API key from platform.claude.com.
  2. On the Workbench, open Ask the Workbench — AI assistant, paste the key, and pick a model (default claude-opus-4-8; Sonnet 4.6, Haiku 4.5, and Fable 5 are also offered).
  3. Tick remember on this device to keep the key in this browser's localStorage for next time, or leave it off to re-enter the key each visit (it then lives only in the page for the session).
  4. Ask away — or use the two one-click features below.
Privacy & CORS Requests go only to api.anthropic.com, sent with the anthropic-dangerous-direct-browser-access: true header that authorises a browser-origin call, plus your x-api-key. Nothing is proxied through this site; the page is static. Your key is used to call Anthropic and is stored only where you choose (session memory, or this device if you tick "remember").

Three ways to use it

  • Chat — ask anything about the reading (“why is the love election red?”, “which planet is strongest?”). With “let Claude run the engine tools” ticked, a tool-using model can compute fresh numbers itself — rank the aims, find the next good window, look up a mansion — and narrate the real result.
  • 📜 Generate the Codex of this Hour — narrates the whole master report tool-by-tool in the grave, image-rich register of Hermes Trismegistus and the Picatrix (yet truthful): for each computation, what it measures, its meaning now, and how the tradition would best use it — with the honest caveat.
  • Plan a working (agentic) — the “conjure rain” box. Type a free-form aim (e.g. “when is the next best time to attempt to call rain, and how would the tradition do it?”). Claude maps it to the nearest catalogued operation, calls the engine tools (findNextElection, rankNow, talismanRecipe) to find the next favourable window, says what to check in the report, and lays out the historical procedure — described, not prescribed. Your requests are remembered for quick reuse.

How the grounding works

The assistant is built on three pure modules so the framing and facts can't drift:

  • assets/js/core/reading.js — fullReading composes the whole engine into one cited object.
  • assets/js/core/llm-context.js — buildContext() builds the locked honest-framing system prompt + the cited fact sheet; toAnthropicTools() exposes the engine's callable capabilities to Claude; runTool() safely runs the real engine for a tool call; buildCodexPrompt() and buildOperationPrompt() are the two task prompts.
  • assets/js/core/registry.js — the capability map the tool schema is derived from.

The local-model option (currently disabled)

The codebase also contains an offline path — a local model via Ollama or an in-browser WebLLM model — so the assistant can run with nothing leaving your machine. That option is turned off in the UI for now while the Claude experience is the focus; it can be re-enabled in assets/js/app/assistant.js. With Ollama you would run ollama serve with OLLAMA_ORIGINS set to allow this page; WebLLM needs WebGPU.
